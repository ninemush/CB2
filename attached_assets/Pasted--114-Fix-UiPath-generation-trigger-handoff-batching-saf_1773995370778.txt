#114 - Fix UiPath generation trigger handoff (batching-safe)
What & Why
The UiPath generation trigger handoff is still failing after the first fix pass. The trigger gets stuck after a single deferred effect evaluation due to React state batching.

Exact root cause: When triggerUiPathGen arrives in the SSE handler, line 1043 calls setPendingUiPathGen(true). When done arrives shortly after (same or next event), line 1051 sets isGeneratingDocRef.current = false but does NOT call setIsGeneratingDoc(false) — only stopDocStreaming() (line 764) does that, and it runs later in the finally block (line 1283). React batches the setPendingUiPathGen(true) from the SSE handler together with the setIsGeneratingDoc(false) from stopDocStreaming in the finally block. The effect at line 1712 watches [pendingUiPathGen, isStreaming, isGeneratingDoc]. It fires once during the batch, sees isGeneratingDoc=true, and logs "effect deferred." Then stopDocStreaming sets isGeneratingDoc=false, but since pendingUiPathGen didn't change again (it was already set to true), React does not reliably re-trigger the effect. The trigger is permanently lost.

Evidence from browser console (runtime):

[UiPath Trigger] triggerUiPathGen received from server
[UiPath Trigger] done received — preserving intent/status for pending UiPath gen
[UiPath Trigger] effect deferred — isStreaming=false, isGeneratingDoc=true, inFlight=false
[UiPath Trigger] finally block — pending trigger detected, will fire via effect
(nothing further — no "effect firing" log, no POST to generate-uipath)

Sending "generate uipath package" reliably triggers generation — POST /api/ideas/:id/generate-uipath appears in server logs every time
The trigger survives React batching — it cannot get stuck after a single deferred effect pass
Console logs show the full lifecycle: trigger request ID created → deferred (with blocking state values) → re-evaluated after blocking clears → consumed (with ID) → fetch started

Progress text shows "Generating UiPath package..." throughout the handoff
