#383 - Stop Emission Gate from destroying XAML declaration infrastructure

What & Why
The Emission Gate scans XML tags in generated XAML and replaces unrecognized ones with Comment+LogMessage stubs. It is currently misclassifying declaration-infrastructure tags such as <AssemblyReference> and <sco:Collection> as activities and destroying them.

These tags are not business activities. They are structural XAML infrastructure used inside:
- TextExpression.ReferencesForImplementation
- TextExpression.NamespacesForImplementation

Once these tags are stubbed out, the declaration blocks are corrupted. The Authoritative Declaration Synthesis step runs afterward, but can no longer find valid insertion anchors, and the Studio Resolution Smoke Test then correctly reports widespread unresolved activities.

Done looks like
Package generation completes without unresolved-activity smoke-test failures caused by declaration-block corruption
The Studio Resolution Smoke Test passes for the previously failing REFramework/infrastructure workflows
<AssemblyReference> and <sco:Collection> are never treated as activities by the Emission Gate
Declaration blocks under TextExpression.ReferencesForImplementation and TextExpression.NamespacesForImplementation survive emission-gate processing intact
Regression tests prove declaration infrastructure is not mutated by the Emission Gate

Out of scope
Refactoring the overall Emission Gate architecture
Changing smoke test logic
Prompt changes
Catalog expansion
Run ID / SSE issues

Tasks
Add "AssemblyReference" and "Collection" to XML_INFRASTRUCTURE_TAGS in emission-gate.ts so they are never treated as activities.

Add "sco" to the structural prefix skip lists used by both:
- enforceActivityEmission
- listActivitiesInBlock

Add a guard so content inside these declaration sections is never activity-enforced:
- TextExpression.ReferencesForImplementation
- TextExpression.NamespacesForImplementation

Add regression tests proving the Emission Gate does not stub or mutate:
- <AssemblyReference>
- <sco:Collection>
- declaration blocks under TextExpression.ReferencesForImplementation
- declaration blocks under TextExpression.NamespacesForImplementation

Verify with an end-to-end package generation that the previously failing REFramework/infrastructure files retain their declaration blocks and the smoke test no longer fails for this corruption reason.
