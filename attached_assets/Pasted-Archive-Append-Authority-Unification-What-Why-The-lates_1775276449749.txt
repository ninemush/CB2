Archive Append Authority Unification

What & Why
The latest build no longer fails on vague post-freeze mutation signals. It now fails at a much tighter boundary: pre-archive integrity at archive append. The error proves that two competing archive-bound content sources still exist after freeze:
- postGateXamlEntries
- deferredWrites

At archive append, these two sources diverge for Main.xaml, violating the single-authoritative-source invariant. This means the pipeline still allows dual content truth at the exact point where archive bytes are assembled.

This task removes that ambiguity. It designates one and only one authoritative workflow-content source for archive-bound XAML at append time, routes archive assembly through that source only, and makes the non-authoritative source derived/read-only or eliminates it from archive-bound use entirely.

Goals
1. Eliminate dual archive-bound workflow content sources at archive append.
2. Ensure archive append consumes exactly one authoritative source of workflow XAML.
3. Make the non-authoritative source read-only or remove it from archive-bound use after freeze.
4. Preserve pre-archive integrity checks as a regression net, but make them pass in normal operation.

Done looks like
- A single explicit authoritative source is chosen for archive-bound workflow XAML at append time.
- Archive append reads workflow bytes only from that authoritative source.
- The non-authoritative source is no longer used to provide archive-bound XAML after freeze.
- If the non-authoritative source still exists, it is derived/read-only after freeze and cannot drift independently.
- Pre-archive integrity verification passes without postGateXamlEntries/deferredWrites divergence in normal operation.
- Any attempt to consult the non-authoritative source for archive-bound workflow bytes after freeze fails immediately with a clear error.
- Build mode behavior is otherwise preserved.

Out of scope
- Broad catalog/filter alignment
- New validator concepts
- Business-logic repair
- Reworking decomposition strategy
- Changing SDD/PDD generation prompts

Tasks

1. Choose the authoritative archive-bound XAML source
Inspect package-assembler.ts and related archive assembly flow and explicitly choose which source is authoritative after freeze:
- postGateXamlEntries
- deferredWrites

The choice must be documented in code near the freeze/append boundary with a clear comment explaining why that source is authoritative.

2. Route archive append through the authoritative source only
Update archive assembly so workflow XAML bytes appended into the archive come from the chosen authoritative source only.
No append path may read workflow bytes from the non-authoritative source after freeze.

3. Make the non-authoritative source derived/read-only
If both structures must still exist for reporting or compatibility reasons:
- the non-authoritative source must be derived from the authoritative one after freeze
- it must become read-only for archive-bound workflow content
- any mutation or independent consultation for archive bytes must fail in package mode

4. Remove append-time dual-source comparison as a producer path
Keep the integrity check, but ensure it only verifies the authoritative source against what is actually appended.
Do not let the integrity check itself depend on two independent writable sources after freeze.

5. Add a strict archive-source guard
Add a guard that throws a package-fatal error if any stage after freeze attempts to:
- append workflow XAML from the non-authoritative source
- mutate the non-authoritative source in a way that could affect archive-bound bytes
- rehydrate archive-bound XAML from stale earlier representations

6. Emit a compact archiveAuthority artifact
Emit a debug artifact such as:
archiveAuthority
with:
- authoritativeSource
- nonAuthoritativeSource
- file
- authoritativeHash
- appendedHash
- identical
- nonAuthoritativeConsultedAfterFreeze: true/false
- violationReason

7. Add regression tests
Cover:
- archive append uses authoritative source only
- non-authoritative source cannot be used for archive-bound bytes after freeze
- non-authoritative mutation after freeze is blocked
- pre-archive integrity passes when only the authoritative source is used
- intentional misuse of the non-authoritative source triggers fatal failure
- normalized filename mapping still works

Relevant files
- server/package-assembler.ts
- server/workflow-status-classifier.ts
- server/final-artifact-validation.ts
- server/uipath-pipeline.ts
- any archive append / deferredWrites helpers
- tests under server/__tests__/
