ISSUE 2 — missing-package-dep must not trigger STUB_BLOCKING_FALLBACK
InitAllSettings is a stub because of missing-package-dep quality 
gate violations. A missing dependency declaration is not a reason 
to destroy a workflow. It is a READY_WITH_WARNINGS condition.

Change the quality gate severity for missing-package-dep from 
blocking error to warning. STUB_BLOCKING_FALLBACK must only trigger 
for: malformed XML, zero activities in a workflow body, or a 
complete generation failure. Not for dependency warnings.
