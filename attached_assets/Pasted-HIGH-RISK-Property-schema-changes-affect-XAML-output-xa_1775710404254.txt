HIGH RISK — Property schema changes affect XAML output:

xaml-template-builder.ts — Builds LLM prompt templates from catalog properties. If xamlSyntax changes from attribute to child-element for an existing activity (e.g., NClick.ClickType), the template will instruct the LLM to emit a completely different XML structure. This is correct long-term but would break deterministic generators that hardcode the old syntax.
deterministic-generators.ts — Hardcodes attribute syntax for NClick, NTypeInto, NGetText, and other N-prefixed activities. If catalog says child-element but generators emit attributes, the validateEmittedActivity check will flag them as violations. These generators must be reconciled with the new property shapes.
required-property-enforcer.ts — Uses required: true to auto-inject missing properties. The DLL extract is conservative (only 6.3% marked required vs the current catalog's higher rate), so this is a reduction in required flags — meaning fewer false-positive injection failures, which is good. But any newly-required properties that weren't required before need review.
type-compatibility-validator.ts — Validates clrType matches between variables and properties. The DLL reveals that many properties we typed as System.String are actually UiPath-specific enums (e.g., UiPath.UIAutomationNext.Enums.NClickType). The validator has no conversion rules for these enum types, so mismatches would be flagged as "unrepairable". For emissionApproved activities, the import must preserve the current simplified clrType values to avoid breaking type compatibility checks.
MEDIUM RISK — Gating and filtering: 5. emission-gate.ts — Checks emissionApproved flag. New activities get emissionApproved: false, so they are automatically blocked from emission. No risk here as long as existing approved activities retain their flag.

filtered-schema-lookup.ts — Gates on emissionApproved. Same as above — new activities are invisible to the generator until explicitly approved. Safe.
prompt-guidance-filter.ts — Only shows packages where generationApproved: true and at least one activity has emissionApproved: true. New packages with all-false emission flags won't appear in prompts. Safe.
LOW RISK — Dependency resolution: 8. post-emission-dependency-analyzer.ts — Resolves activities to NuGet packages. New catalog entries improve resolution accuracy. The catalog-first lookup means new packages will be found without needing PACKAGE_NAMESPACE_MAP updates. However, if any new emissionApproved: true activity gets emitted and its package lacks an xmlns mapping in xaml-compliance.ts, namespace injection will fail. Safe as long as new activities stay emissionApproved: false.

uipath-activity-registry.ts — Proxy merges catalog with fallback registry. More catalog entries = better resolution. No regression risk.
