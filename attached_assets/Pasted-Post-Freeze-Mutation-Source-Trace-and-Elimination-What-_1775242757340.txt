Post-Freeze Mutation Source Trace and Elimination

What & Why
The latest build now fails with a stronger integrity signal:
the finalized archive buffer diverges from the frozen content after the classifier freeze point, with multiple file hash mismatches. This proves there is still at least one late-stage mutation path rewriting archive-bound XAML after freeze.

This is no longer mainly a classification or DHG parity problem. The current problem is that frozen archive-bound content is still being mutated afterward.

This task must identify every actual post-freeze mutation source, trace exactly where the bytes change, and eliminate or relocate those mutations so archive-bound XAML is truly immutable after freeze.

Goals
1. Identify the exact stage(s), function(s), and mutation reason(s) that alter frozen archive-bound XAML after freeze.
2. Eliminate or relocate those mutations to before freeze.
3. Enforce that no archive-bound bytes can change after freeze.
4. Preserve the current fatal integrity checks, but make them unnecessary in normal operation because no post-freeze mutation occurs.

Done looks like
- For every frozen workflow file, the system can prove whether any post-freeze mutation was attempted.
- The exact source of each mutation is identified with stage, function, file, pre-hash, post-hash, and reason.
- All currently allowed late-stage mutations are either:
  - moved to before freeze, or
  - removed entirely in package mode.
- The finalized archive buffer matches frozen content hashes for all workflows.
- Post-archive freeze verification passes without mismatch.
- Build mode remains preserved except that post-freeze mutation is now impossible in package mode.

Out of scope
- Broad catalog/filter alignment
- New validator concepts
- Business-logic repair
- Reworking decomposition strategy
- Changing SDD/PDD generation prompts

Tasks

1. Add mutation provenance tracing
Instrument all code paths after freeze that can touch:
- deferredWrites
- archive buffers
- workflow maps
- DHG workflow inputs
- final reporting workflow inputs
- any XAML string mutation helper

For each attempted mutation, record:
- stage
- function
- file
- preHash
- postHash
- mutationReason
- stack/context label if available
- allowed: true/false

2. Emit a structured artifact
Emit:
`postFreezeMutationTrace`
with:
- per-mutation records
- per-file mutation counts
- compact summary:
  - totalAttemptedMutations
  - totalBlockedMutations
  - totalUnexpectedPostFreezeMutations
  - filesChangedAfterFreeze

3. Identify all post-freeze mutators
Specifically inspect for late-stage mutation in or after:
- archive append / deferredWrites flush
- DHG generation inputs
- final artifact validation
- parity verification helpers
- any repair/stub/placeholder injection path
- any quote-fix / expression-fix / normalization helper
- any project.json or manifest assembly path that may rehydrate workflow content

4. Move or eliminate mutations
For every mutation currently happening after freeze:
- move it before freeze if it is a legitimate canonicalization/normalization step
- otherwise remove it from package mode
- no post-freeze mutation may remain for archive-bound workflow bytes

5. Harden freeze enforcement
After freeze, replace mutable structures with guarded/frozen wrappers where practical.
Any attempt to mutate frozen workflow content in package mode must fail immediately with a stage-attributed error before archive finalization.

6. Add regression tests from the current failure pattern
Cover:
- late mutation after freeze on Finalize.xaml
- late mutation after freeze on Main.xaml
- multiple files changing after freeze
- deferredWrites / archive-buffer mutation after freeze
- DHG or reporting stage attempting to rewrite workflow bytes after freeze
- successful run where frozen hashes equal archived hashes for all files
