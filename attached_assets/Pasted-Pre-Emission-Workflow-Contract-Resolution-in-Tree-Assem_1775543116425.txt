Pre-Emission Workflow Contract Resolution in Tree Assembler

What & Why
The latest authority-gap assessment identified workflow contract authority as the weakest remaining architectural gap. Today, caller/callee contracts are validated only after emission, while InvokeWorkflowFile bindings are emitted earlier without authoritative knowledge of the callee’s declared arguments. This means the pipeline still discovers contract truth too late. The post-emission validator and canonicalizer are acting as safety nets for defects that should never have been emitted in the first place.  [oai_citation:0‡authority-gap-assessment.md](sediment://file_00000000b39872468c7feaf945099d10)

This task makes workflow contract truth available before per-workflow assembly begins, so the tree assembler can emit InvokeWorkflowFile bindings against known callee contracts on the first pass.

Goal
Make the tree assembler aware of callee argument contracts before it emits InvokeWorkflowFile bindings, so it only emits structurally valid bindings in the correct direction and form.

Done looks like
- A contract map is built from the available workflow specs before per-workflow assembly begins.
- When the tree assembler emits an InvokeWorkflowFile, it looks up the callee contract first.
- Only bindings supported by the callee contract are emitted.
- Argument direction is emitted correctly using the known contract truth rather than inferred after the fact.
- Invoke bindings are emitted in the correct structured form on first pass.
- unknown_target_argument and invoke_argument_binding_mismatch defects drop materially on the benchmark use cases.
- Post-emission contract validation and invoke canonicalization remain in place as safety nets, but they are no longer the primary mechanism making invoke contracts coherent.

Out of scope
- Changing the enrichment pipeline or LLM prompts
- Rewriting workflow-contract-integrity.ts or invoke-binding-canonicalizer.ts as new primary systems
- Broad activity schema enforcement changes
- Expression authority changes
- REFramework wiring redesign beyond what is strictly necessary for contract-aware invoke emission
- Silent synthesis of missing contract truth when the workflow spec does not support it

Tasks
1. Build a pre-assembly workflow contract map from the available workflow specs.
   The contract map should provide, for each workflow, the declared arguments with:
   - argument name
   - direction
   - type, where available from the spec
   - enough shape information for the tree assembler to emit valid InvokeWorkflowFile bindings

2. Pass the contract map into the tree assembly flow before per-workflow XAML emission begins.

3. Update the InvokeWorkflowFile emission path in workflow-tree-assembler.ts so that it:
   - resolves the callee first
   - looks up the callee contract in the prebuilt contract map
   - emits only bindings that are valid for that callee
   - emits them in the correct structured child-element form
   - uses the contract truth for direction handling rather than leaving correction to post-emission repair

4. Add a strict evidence rule for unsupported bindings.
   If the workflow spec provides a binding that does not exist in the callee contract map, do not silently treat it as valid.
   Instead:
   - withhold the unsupported binding from emission
   - record an explicit contract diagnostic explaining what was rejected and why
   - do not silently invent missing contract truth

5. Keep post-emission systems as safety nets.
   Do not remove:
   - workflow-contract-integrity.ts
   - invoke-binding-canonicalizer.ts
   They should remain in place for audit and residual edge-case repair, but this task should reduce how often they need to intervene.

6. Add focused regression tests covering:
   - contract map construction from workflow specs
   - InvokeWorkflowFile emission using a known callee contract
   - unsupported binding withheld with explicit diagnostic
   - correct direction/form emission on first pass
   - post-emission validator still passes valid benchmark-style invokes without needing major canonicalization
   - no silent invention of missing contract truth

Implementation constraints
- Prefer the smallest authority-boundary change that moves contract truth from post-emission audit to pre-emission assembly.
- Do not broaden into a combined workflow-contract plus expression plus schema mega-task.
- Do not guess contract details that are not supported by the workflow specs.
- If the current workflow spec data is insufficient for some bindings, surface that honestly in diagnostics rather than fabricating structure.

Likely primary files to touch
- server/workflow-tree-assembler.ts
- server/package-assembler.ts
- server/xaml/workflow-contract-integrity.ts
- server/xaml/invoke-binding-canonicalizer.ts
- server/xaml-generator.ts
- server/final-artifact-validation.ts

What this does not touch
- LLM prompt changes
- enrichment model redesign
- symbol authority work already covered by the active symbol task
- broad activity schema timing fixes
- broad expression builder redesign

Success measure
The benchmark packages should show a clear reduction in contract-integrity failures caused by emitting invoke bindings before callee truth is known, and the emitted invoke structure should be correct on first pass rather than repaired later.
