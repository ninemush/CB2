After a server restart, a run record can remain in status: running even though the in-memory execution is gone. That blocks all new generation attempts until the stale threshold expires.

Observed failure:
- previous run died during/after server restart
- DB row remained status: running
- a new generation attempt was blocked by checkDbInProgress
- the 10-minute stale threshold treated the dead run as alive
- user is locked out even though no real run exists anymore

Fix the run-lifecycle ownership model so server restarts and orphaned runs are handled correctly.

   - Do not leave pre-restart in-progress runs blocking new work
